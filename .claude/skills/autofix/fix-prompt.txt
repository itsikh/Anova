## Project Structure

- `app/src/main/java/com/template/app/` - Main source
  - `anova/` - Core device layer
    - `AnovaProtocol.kt` - BLE UUIDs (FFE0/FFE1), ASCII command constants
    - `AnovaDeviceState.kt` - ConnectionState, TempUnit, AnovaStatus enums + AnovaDeviceState data class
    - `AnovaRawState.kt` - Shared poll result returned by all transports
    - `ConnectionMode.kt` - BLUETOOTH / LOCAL_WIFI / CLOUD / AUTO enum
    - `ThresholdSettings.kt` - Min/max temperature alert thresholds
    - `AnovaTransport.kt` - Interface: poll(), connect(), disconnect(), connectionState StateFlow
    - `AnovaBluetoothManager.kt` - BLE transport: scans for "Anova" device, GATT FFE0/FFE1 profile
    - `AnovaWifiTransport.kt` - TCP socket transport to device IP:8080, same ASCII protocol
    - `AnovaSettings.kt` - DataStore-backed settings (connection mode, IP, email, poll intervals)
    - `AnovaRepository.kt` - Orchestrates transports, auto-fallback (WiFi→Cloud 6s timeout), polls, stores history
    - `cloud/AnovaCloudConfig.kt` - Firebase API key, sign-in/refresh URLs, Anova base URL
    - `cloud/AnovaCloudModels.kt` - Firebase auth response, device list, device state response models
    - `cloud/AnovaFirebaseAuth.kt` - Token cache + refresh logic
    - `cloud/AnovaCloudTransport.kt` - Cloud transport: Firebase auth → /devices → /cooker/{id}/state
  - `history/` - Room DB
    - `TemperatureReading.kt` - @Entity: timestamp, temperature, unit, status, timerMinutes
    - `TemperatureReadingDao.kt` - insert, getRecentReadings (Flow), getReadingsInRange, deleteOlderThan
    - `AnovaDatabase.kt` + `AnovaDatabaseModule.kt` - Room DB singleton + Hilt module
  - `notifications/AnovaAlertManager.kt` - Posts high-priority notifications on threshold breach
  - `ui/screens/monitor/`
    - `AnovaViewModel.kt` - @HiltViewModel: deviceState, activeTransport, thresholds, connect/disconnect
    - `MonitorScreen.kt` - 4-mode chip selector, temperature display, threshold chips, connect button
    - `ConnectionSettingsDialog.kt` - WiFi/cloud/BLE config dialog
    - `ThresholdDialog.kt` - Min/max threshold toggles + fields
  - `ui/screens/history/`
    - `HistoryViewModel.kt` - Provides readings StateFlow (200 most recent)
    - `HistoryScreen.kt` - Canvas line chart + LazyColumn reading list
  - `ui/screens/settings/` - SettingsScreen, SettingsViewModel (GitHub token, backup, debug)
  - `ui/screens/bugreport/` - BugReportScreen, BugReportViewModel
  - `ui/navigation/AppNavHost.kt` - Routes: monitor (start), history, settings, bug_report/{mode}
  - `ui/components/FloatingBugButton.kt` - Draggable overlay bug button (awaitEachGesture tap/drag)
  - `backup/AnovaBackupManager.kt` - Extends BaseBackupManager; exports/imports temperature history as JSON
  - `AppConfig.kt` - Central config (GitHub: itsikh/Anova for issues + releases)
  - `TemplateApplication.kt` - @HiltAndroidApp, notification channels, crash handler
  - `MainActivity.kt` - Single activity, injects CrashAutoReporter
- `app/build.gradle.kts` - Build config (versionCode, versionName, signing, deps)
- `app/src/main/AndroidManifest.xml` - BLE permissions, INTERNET, POST_NOTIFICATIONS, etc.
- Key technical facts:
  - BLE protocol: write ASCII command + \r to FFE1, read response from FFE1 notification
  - WiFi protocol: TCP socket to device IP:8080, same ASCII commands
  - Cloud protocol: unofficial Firebase auth → oven.anovaculinary.com REST API
  - AUTO mode: try WiFi first (6s timeout) → fallback to Cloud
  - Poll intervals: local=2s (configurable), remote=60s (configurable), persisted in DataStore
  - Temperature history: 30 days max, stored in Room, exported in backup
  - Cloud password stored in EncryptedSharedPreferences via SecureKeyManager
